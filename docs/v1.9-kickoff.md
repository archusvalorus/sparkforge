# Sparkforge v1.9 — "The Deepening" Sprint Kickoff

*Scoped July 15, 2026, immediately after v1.8 "The Mirrorwound" entered App
Store review. A deliberate **systems-only** version — the foundation that
makes every future content beat cheap and cohesive. No new arena; the game
gets deeper, not wider. DoD gates between units stay absolute; Brandon owns
the interactive playthrough gate — and v1.9 is as much a **balance marathon**
as a build sprint.*

**Thesis:** Cards stop being one-and-done. They gain **power tiers** (a
mini-progression inside the run), so a card becomes a mini-tree instead of us
inventing new cards forever. Level-ups reward you **every** level (a skill
card always; stats chosen on even levels, random on odd), and a live **stat
HUD** makes that growth legible while you choose. This is the version that
turns the card system into a *factory* — after it, a themed arena is just
hazards + mobs + boss + theme + tier-aware cards run through the pipeline.

## Why systems, not content (and why that's not a violation of content > systems)

- **Card leveling is a massive player-facing change**, not backend plumbing —
  "every card grows; every run is a new build" is felt harder than a new arena.
- **The systems ARE content infrastructure.** v1.9 is the factory; v2.0+ is the
  assembly line (themed arenas generated cohesively and cranked through the
  established build→gate workflow).
- **Right time to stress-test.** Brandon has a clean before/after read on the
  compounding systems now, before the player base expands.

## The road (arc context — not v1.9 scope)

- **v1.9 "The Deepening"** (this doc): card leveling engine + tier ladders,
  the every-level cadence, the stat HUD.
- **v2.0 "Arena 5 + The Mote"** (`docs/v2.0-outline.md`): the coupled finale —
  Arena 5 → the Mote entrance one-shot → the earned Spark skin, plus Boss Mode
  (cut-line) + a tuning pass. The comfortable walk-away point.
- **v2.1+ / X.X.5** (banked): themed-zone arena geometry, then procedural
  layouts, more tier ladders (e.g. the attack-speed laser capstone).

## Decided — do not relitigate

- **v1.9 is systems-only. No new arena, no new enemies/boss.** Content rides
  in v2.0. (See the reframe above — this is deliberate, not a compromise.)
- **Card leveling = power TIERS per card.** Picking a card you already own
  **levels it up** instead of being excluded. Max tier adds a qualitative
  **capstone**.
  - **Tier matrix: cards are 1 / 2 / 3 / 5-tier. NO 4-tier.** Each ability is
    judged for how many tiers it can carry while staying fun/meaningful.
  - **Orthogonal to synergies.** Breadth (distinct cards per tree) feeds
    synergy tiers (3/5/7); depth (card tiers) is its own axis. The tension —
    deepen vs. spread — is the point.
  - **One tier-5 capstone card per skill tree.** Bounds the chaos.
  - **Per-run** (like `appliedSynergies` — zero save-data risk). Tiers open in
    the Codex as they're reached.
- **Backward-compat is the de-risk.** A card's effect becomes a tier-ladder;
  a one-rung ladder = a 1-tier card = *identical to today* (maxed after one
  pick → excluded from re-draw). Existing cards keep working untouched;
  laddering is opt-in per card. So v1.9 ships the **engine + signature
  ladders**, and the full ~65-card retrofit proceeds **incrementally** — no
  content wall at launch.
- **Level-up cadence (the anchor):**
  - **Every level → a skill (tree) card pick.**
  - **Even levels → a stat CHOICE first** (pick +HP/+ATK/+DEF), then the skill
    card (two picks).
  - **Odd levels → random auto-awarded stats** (+HP/+ATK/+DEF, not chosen)
    alongside the skill card.
  - Scarcity for card leveling now comes from total level count + deepen-vs-
    spread + the capstone cap (NOT "cards only on odd").
- **Stat HUD ships with the cadence rework** — a right-side HP/ATK/DEF box so
  chosen and random stat gains are legible during choices. One package.
- **Hand Lyra the Arena 5 brief now** (with the tier rules), so v2.0 content
  is ready to build the moment v1.9 ships. Lyra-facing work parallelizes with
  the v1.9 engine build.

## Unit sequence

### Unit 1 — Card-leveling engine (backward-compatible)
- `drawCards` re-offers owned, non-maxed cards (today it hard-excludes all
  owned). Maxed cards still excluded.
- `apply: (PlayerStats) -> Void` → a **tier-ladder** (ordered per-tier delta
  closures; a pick runs the next rung). PlayerStats is all-additive, so each
  rung applies its own delta. A one-rung ladder = today's behavior exactly.
- Per-card tier state (per-run); card-selection UI shows current tier +
  next-tier preview.
**DoD:** owned non-maxed cards re-appear and level up; 1-tier cards behave
exactly as v1.8 (regression-free); tier + next-tier legible on the card; no
save-data change.

### Unit 2 — Codex tier absorption
- The Codex / `CardDetailNode` shows a card's tier ladder and the max tier
  reached. "Discovered = ever offered" semantics unchanged.
**DoD:** tiered cards render their ladder in the Codex; undiscovered/partial
states correct; no schema churn (v1.8's card codex design absorbs it).

### Unit 3 — Signature ladders + tier-matrix pass
- Assign the 1/2/3/5 tier to a starter set of **signature cards** and author
  their ladders — including the canonical multishot ladder and **one tier-5
  capstone per tree**. Remaining cards stay 1-tier (backward-compat) and get
  laddered incrementally in later passes.
- Lyra-designed where creative (capstone identities); mechanically true.
**DoD:** signature ladders in pool and mechanically true at every tier; one
capstone per tree; capstones read as distinct from synergy tiers; tag counts
still ≥7.

### Unit 4 — Level-up cadence rework
- Every level presents a skill card. **Even** levels present a stat-choice
  selection first (pick +HP/+ATK/+DEF), then the skill card. **Odd** levels
  auto-award a random stat alongside the skill card. Reuse the level-up
  overlay + the sequential-modal pattern.
- **Balance-sensitive:** re-tunes the run power curve; stacks with card
  leveling + synergies. Tune against the stable post-v1.8 baseline.
**DoD:** cadence correct (every level a card; even = stat choice + card; odd =
random stat + card); every level rewards; feel + balance validated on device.

### Unit 5 — Playfield stat HUD
- Right-side HP/ATK/DEF box, clear of existing HUD (timer/XP/HP-bar top-left,
  buff badges left, pause top-right, in-run Remove-Ads upper-left). Live
  updates so chosen + random stat gains are readable during choices.
**DoD:** HP/ATK/DEF legible and live; no HUD collisions; readable during the
even-level stat choice; 60fps held.

### Unit 6 — Forge XP curve + level-up correctness
- `xpForLevel` scaling **1.5 → 1.35** (`ProgressionManager`) — a gentler
  high-level curve so a high-volume player's level tracks their play. Forge
  Level stays uncapped; it's a fun long-tail power-augmentation lane, not a
  wall. (Later levels may not award more forge XP unless we tune them to.)
- `addForgeXP`: **`if` → `while`** — a run now grants ALL the levels its XP
  earned, not just one. Latent correctness bug; self-corrected before but wrong.
**DoD:** curve at 1.35; a multi-level run pops multiple levels in one go;
existing `sf_forge_xp` / `sf_forge_level` preserved; no key changes.

### Unit 7 — Forge Path management screen + free respec (was v1.8 Unit 10b, cut)
- Make the Forge Path entry **reviewable ANY time**, not only when picks are
  available (today [TitleScene] guards `picksAvailable > 0` and dead-taps once
  spent). A dedicated screen: all three branches (Vitality/Ferocity/Cunning),
  current picks per branch, each branch's next node, and spend-here.
- **FREE & UNLIMITED respec** (pre-decided in v1.8 Unit 10b) behind a confirm
  ("Refund all N picks to re-spend?"). New `ForgePathManager.respec()` clears
  picks; `applyPathBonuses` unchanged; keys stay `sf_forge_path_picks` (never
  rename — live data).
- A tappable entry around the Forge LV area to open it; coordinate with the
  existing `forgePathButton` and the mis-tap-into-run concern.
- **OPEN (Brandon's tentative idea):** rename "Forge Path picks" → **"Mastery
  points"** in UI copy (keys unchanged). Decide during the UI pass.
**DoD:** screen opens any time (never starts a run); branches/picks/next-node
legible; respec refunds free behind a confirm and re-spend works; run stats
apply; no ignite-CTA regression; no key rename. Device-validated.
