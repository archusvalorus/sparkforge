### Notable fixes worth remembering
- **Monument bosses broke every range check** — auto-aim measured to the boss *origin*; a monument's body radius is `arenaRadius × 0.40`, so a player touching it was "far away" and auto-aim refused to fire. Fixed via `targetingRadius` on `ArenaBossNode` (default 0), applied to auto-aim, the Apex familiar, and Skybeam's lasso.
- **Mote's punchline never fired** — the `MOTE FOUND YOU` override had been written into the death-overlay *construction* function instead of `playerDied()`, so it evaluated once at setup when the flag was false.

## NEXT: Unit B2 — the Gauntlet

**Boss Mode is NOT a practice selector.** It's **one button → bosses back-to-back
until you clear them all or die.** (The per-boss practice pick built in B1 stays
as a liked secondary layer.)

### Design — all settled, all config-driven

- **Order:** Sequential (arena order) and Mixup (random, capped). `BossRegistry.sequential` / `.mixup(count:)` already return these.
- **Arenas:** each boss fights in its **own home arena** — no adds (normal play already spawns nothing while a boss lives). A **Mashup** mode (boss × foreign arena) is a LATER feature.
- **Opener:** **3 auto-granted cards** ("as if you leveled up 3 times"). Model is **ARAM** — start ahead, no menu friction, one button straight in. Grants, *not* picks.
- **Progression:** level as you go. Bosses award **5× XP**, growing **+1× per stage** (stage 1 = 5×, stage 2 = 6× …).
- **Difficulty ramp:** arena bosses `+12.5% × stage`. **Monuments are DECOUPLED** — flat, stage-independent, **start at +25%**.
  - *Reasoning:* the ramp does **not** compound (each buff applies to that boss's own base). But a monument's base is already multiples of an arena boss, so handing it the biggest multiplier makes a **cliff, not a curve**. And bias the wall **LOW**: an unbeatable final boss silently kills the mode, while a too-easy one just gets buffed next patch. Asymmetric risk.
- **Heals:** **2–3 HP orbs, randomized, after each boss.** (Maybe 2–4 later; the variance is the point.)
- **Death = run over.** **No ads, no revives.** "It's pure boss mode."
- **Rewards:** sandbox-isolated. **Forge XP capped at 100 per gauntlet run** — uncapped makes Boss Mode the optimal farm and warps normal play; zero makes it pointless.

### Suggested split
- **B2a** — the gauntlet loop: state machine, stage transitions, arena swapping mid-run, boss handoff. Playable end-to-end, sequential only.
- **B2b** — the 3-card opener, inter-boss heals, XP/ramp tuning, capped Forge XP, Mixup.
- **B3** — challenge dials (HP/ATK/DEF, up *and* down) + run summary.
- **B4** — the 5-boss tuning pass, as a set.

### Build notes for whoever picks this up
- Most of the complexity lives in **`GameScene`'s run lifecycle** (`restartGame`, `playerDied`, the spawn dispatch around `spawnEvent.shouldSpawnMiniBoss`, and `setupArena`). Re-read those before writing the loop rather than half-remembering them.
- `ArenaConfig.current` is driven by `ProgressionManager.currentArena` and clamped by `arenasUnlocked` — arena swapping mid-gauntlet has to work *with* that, not around it.
- `GameConfig.Arena.radius` folds in `radiusScale` per-arena, so swapping arenas mid-run changes the field size. The monument anchors off that radius.
- **Arena 5 is DEBUG-only for real players right now** — there's a `#if DEBUG` unlock in `TitleScene.didMove`, and the clear-Arena-4 → unlock-Arena-5 wiring is deliberately NOT live until Unit 2 fully ships.
